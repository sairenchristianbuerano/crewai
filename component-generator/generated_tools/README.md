# Generated Tools Directory

This directory contains CrewAI tools that have been generated by the Component Generator service.

## Purpose

When you generate a tool using the API endpoints:
- `POST /api/crewai/tool-generator/generate`
- `POST /api/crewai/tool-generator/generate/sample`

The generated Python code is automatically saved to this directory for:
- **Testing**: Easily test the generated tools locally
- **Reference**: Review the generated code structure
- **Debugging**: Inspect generated code if issues occur
- **Learning**: See examples of generated tools

## File Naming

Generated files are named: `{ToolName}.py`

Example:
- `CustomApiTool.py`
- `PlaygroundCalculator.py`
- `WebScraperTool.py`

## Viewing Generated Code

Generated code is also logged to the console/docker logs with separators:

```
================================================================================
Generated CustomApiTool.py:
================================================================================
[Full Python code here]
================================================================================
```

## Git Ignore

This directory is excluded from git (see `.gitignore`) to prevent committing:
- Test outputs
- Temporary generated files
- Experimental tool code

## Cleanup

You can safely delete files in this directory. They will be regenerated on the next API call.

```bash
# Clean all generated tools
rm -rf component-generator/generated_tools/*.py

# Keep only README
find component-generator/generated_tools -name "*.py" -delete
```

## Docker Volume Mount

In `docker-compose.yml`, this directory is mounted to `/app/data/generated_tools` inside the container:

```yaml
volumes:
  - ./component-generator/generated_tools:/app/data/generated_tools
```

This allows generated files to persist on your local filesystem even when containers are stopped/removed.
